# 密码生成器网站 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位

一个安全、易用的在线密码生成工具，帮助用户快速生成强密码，提升账户安全性。

### 1.2 产品目标

- 提供安全可靠的密码生成服务
- 支持多种密码生成规则和自定义选项
- 提供良好的用户体验和直观的界面
- 确保所有密码生成在客户端完成，保护用户隐私

### 1.3 目标用户

- 需要创建新账户的用户
- 需要定期更换密码的用户
- 对账户安全有较高要求的用户
- 需要批量生成密码的开发者或管理员

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 密码生成

- **基础生成**：支持生成指定长度的随机密码
- **字符类型选择**：
  - 大写字母 (A-Z)
  - 小写字母 (a-z)
  - 数字 (0-9)
  - 特殊字符 (!@#$%^&\*等)
- **密码长度设置**：可自定义密码长度（建议范围：8-128 字符）
- **排除相似字符**：可选排除容易混淆的字符（如 0/O, 1/l/I）
- **排除歧义字符**：可选排除容易产生歧义的字符

#### 2.1.2 密码显示与复制

- 实时显示生成的密码
- 一键复制密码到剪贴板
- 复制成功后提供视觉反馈
- 密码显示支持显示/隐藏切换

#### 2.1.3 密码强度评估

- 实时显示密码强度（弱/中/强/极强）
- 基于长度、字符类型多样性、随机性等因素评估
- 提供强度可视化指示器

#### 2.1.4 批量生成

- 支持一次生成多个密码
- 可设置生成数量（如 5、10、20 个）
- 批量密码列表展示
- 支持批量复制或单独复制

### 2.2 高级功能

#### 2.2.1 密码预设模板

- 常用密码类型预设：
  - 简单密码（仅字母和数字）
  - 强密码（包含所有字符类型）
  - 超强密码（长密码 + 所有字符类型）
  - 记忆友好密码（可读性更好的密码）
- 自定义保存常用配置

#### 2.2.2 密码历史记录

- 本地存储最近生成的密码（可选）
- 支持查看和重新复制历史密码
- 提供清除历史功能
- 注意：历史记录仅存储在本地，不上传到服务器

#### 2.2.3 密码导出

- 支持将生成的密码导出为文本文件
- 支持导出为 CSV 格式（适用于批量密码）

### 2.3 辅助功能

#### 2.3.1 密码建议

- 根据使用场景提供密码建议（如：网站登录、WiFi 密码、API 密钥等）
- 显示密码最佳实践提示

#### 2.3.2 安全提示

- 提醒用户不要重复使用密码
- 建议定期更换密码
- 提醒使用密码管理器

## 3. 非功能需求

### 3.1 性能要求

- 页面加载时间 < 2 秒
- 密码生成响应时间 < 100ms
- 支持生成大量密码时保持流畅

### 3.2 安全要求

- **客户端生成**：所有密码在用户浏览器中生成，不经过服务器
- **无数据收集**：不收集、不存储、不上传任何用户数据
- **加密传输**：使用 HTTPS 协议
- **随机性保证**：使用加密安全的随机数生成器（Crypto API）

### 3.3 兼容性要求

- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 支持移动端浏览器（响应式设计）
- 支持现代浏览器（ES6+）

### 3.4 可用性要求

- 界面简洁直观，易于理解
- 支持键盘快捷键操作
- 提供清晰的操作反馈
- 支持暗色/亮色主题切换

### 3.5 可访问性要求

- 支持屏幕阅读器
- 符合 WCAG 2.1 AA 标准
- 支持键盘导航

## 4. 用户故事

### 4.1 核心用户故事

**作为** 一个需要创建新账户的用户  
**我希望** 能够快速生成一个强密码  
**以便** 保护我的账户安全

**作为** 一个需要定期更换密码的用户  
**我希望** 能够自定义密码规则  
**以便** 满足不同网站的要求

**作为** 一个开发者  
**我希望** 能够批量生成密码  
**以便** 用于测试或初始化账户

**作为** 一个对隐私敏感的用户  
**我希望** 密码在本地生成  
**以便** 确保我的密码不会被泄露

### 4.2 高级用户故事

**作为** 一个经常使用密码生成器的用户  
**我希望** 能够保存常用的密码配置  
**以便** 快速生成符合特定要求的密码

**作为** 一个需要管理多个密码的用户  
**我希望** 能够查看最近生成的密码  
**以便** 在需要时重新使用

## 5. UI/UX 设计要求

### 5.1 界面布局

- **顶部区域**：标题和主要说明
- **中央区域**：密码生成配置面板和生成的密码显示区
- **底部区域**：操作按钮和辅助信息

### 5.2 视觉设计

- 现代化、简洁的设计风格
- 清晰的视觉层次
- 合适的颜色对比度
- 支持暗色/亮色主题

### 5.3 交互设计

- 实时预览密码生成结果
- 即时反馈用户操作
- 流畅的动画过渡
- 清晰的错误提示

### 5.4 响应式设计

- 桌面端：充分利用屏幕空间
- 平板端：适配中等屏幕
- 移动端：单列布局，易于触控操作

## 6. 技术需求

### 6.1 前端技术栈（建议）

- **框架**：React / Vue.js / 原生 JavaScript
- **样式**：CSS3 / Tailwind CSS / Styled Components
- **构建工具**：Vite / Webpack
- **密码生成**：Web Crypto API

### 6.2 核心算法

- 使用 `crypto.getRandomValues()` 生成加密安全的随机数
- 实现字符池随机选择算法
- 密码强度评估算法

### 6.3 数据存储

- 使用 localStorage 存储用户偏好设置
- 可选：使用 IndexedDB 存储密码历史（如果实现）

### 6.4 部署要求

- 静态网站托管（如 GitHub Pages、Netlify、Vercel）
- HTTPS 支持
- CDN 加速

## 9. 风险评估

### 9.1 技术风险

- **随机数生成质量**：确保使用加密安全的随机数生成器
- **浏览器兼容性**：某些旧版浏览器可能不支持 Web Crypto API

### 9.2 安全风险

- **密码泄露**：确保所有操作在客户端完成，不经过服务器
- **XSS 攻击**：需要做好输入验证和输出转义

### 9.3 用户体验风险

- **密码过于复杂**：提供不同复杂度的选项
- **用户误操作**：提供清晰的提示和确认机制
